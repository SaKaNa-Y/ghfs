# GHFS0302: Cannot remove a reaction without an authenticated user

Code: `GHFS0302`
Level: error

## What this error means

A `remove-reaction` op was applied while ghfs couldn't resolve the GitHub user the token authenticates as. GitHub's REST API removes reactions by `reaction_id`, which is per-user, so ghfs needs to know *whose* reaction to delete.

## Why this happens

- `GH_TOKEN` / `GITHUB_TOKEN` is not set
- The token is invalid or expired
- `gh auth login` hasn't been run in this environment

## How to fix it

Set a valid token before running execute:

```bash
export GH_TOKEN=ghp_xxx
ghfs execute --run
```

Or sign in with `gh`:

```bash
gh auth login
```

## Additional context

- Reactions on PR review summaries go through GraphQL and use the viewer implicitly, so this error only applies to issue / comment reactions
- `add-reaction` doesn't require authenticated-user lookup (GitHub attributes the new reaction to the token's owner automatically)
